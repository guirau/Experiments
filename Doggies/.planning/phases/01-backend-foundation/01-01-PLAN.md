1. Create `v1/backend/config.py`:
```python
import os
from dotenv import load_dotenv

# Load .env from the backend directory
load_dotenv(dotenv_path=os.path.join(os.path.dirname(__file__), ".env"))


def _require(name: str) -> str:
    value = os.environ.get(name)
    if not value:
        raise RuntimeError(f"Missing required env var: {name}")
    return value


class Settings:
    SUPABASE_URL: str = ""
    SUPABASE_ANON_KEY: str = ""

    def __init__(self) -> None:
        self.SUPABASE_URL = _require("SUPABASE_URL")
        self.SUPABASE_ANON_KEY = _require("SUPABASE_ANON_KEY")


settings = Settings()
```

Note: Using plain python-dotenv (not pydantic-settings) per D-03 (requirements.txt, minimal deps).

2. Create `v1/backend/database.py`:
```python
import logging
from supabase import Client, create_client
from config import settings

logger = logging.getLogger(__name__)

_client: Client | None = None


def get_supabase_client() -> Client:
    global _client
    if _client is None:
        _client = create_client(settings.SUPABASE_URL, settings.SUPABASE_ANON_KEY)
        logger.info("Supabase client initialized")
    return _client
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem → config.py | .env file is read from disk; if the file is world-readable, credentials leak |
| config.py → process environment | Settings constructor reads env vars at import time |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | v1/backend/.env | mitigate | Add `.env` to `.gitignore` immediately in Task 2; `.env.example` carries no real credentials |
| T-01-02 | Denial of Service | config.py Settings.__init__ | accept | Fail-fast RuntimeError on missing env vars is intentional — prevents silent misconfiguration; error message does not print credential values |
| T-01-03 | Information Disclosure | database.py logger | accept | Logger only emits "Supabase client initialized" — no URL or key values in log output; low risk for local dev tool |
</threat_model>
