```python
import logging
from typing import Any

from fastapi import APIRouter, HTTPException
from database import get_supabase_client

logger = logging.getLogger(__name__)

router = APIRouter()

# Fields returned by GET /api/dogs/{id} — excludes unused fields per PRD §4
DOG_DETAIL_FIELDS = (
    "id, name, breed, age_estimate, size, gender, "
    "temperament, medical_notes, story, photos, status, intake_date"
)


@router.get("/dogs")
def list_dogs() -> dict[str, list[dict[str, Any]]]:
    """GET /api/dogs — returns { dogs: [{ id, name }] }"""
    client = get_supabase_client()
    response = client.table("dogs").select("id, name").execute()
    return {"dogs": response.data}


@router.get("/dogs/{dog_id}")
def get_dog(dog_id: str) -> dict[str, Any]:
    """GET /api/dogs/{id} — returns full dog object or 404"""
    client = get_supabase_client()
    response = (
        client.table("dogs")
        .select(DOG_DETAIL_FIELDS)
        .eq("id", dog_id)
        .execute()
    )
    if not response.data:
        raise HTTPException(status_code=404, detail=f"Dog {dog_id} not found")
    return response.data[0]
```

Note on .eq("id", dog_id): Supabase-py `.eq()` filters by equality. If `dog_id` is not a valid UUID, Supabase will return an empty list (not an error), so the 404 path handles malformed UUIDs too.

```python
import logging

from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware

from routers.dogs import router as dogs_router

logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s [%(levelname)s] %(name)s: %(message)s",
)

logger = logging.getLogger(__name__)

app = FastAPI(title="Doggies API", version="1.0.0")

# CORS — allow requests from the Next.js dev server (Phase 2)
# Also allow all origins for local development convenience
app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:3000", "*"],
    allow_credentials=False,
    allow_methods=["GET"],
    allow_headers=["*"],
)

app.include_router(dogs_router, prefix="/api")


@app.get("/health")
def health() -> dict[str, str]:
    return {"status": "ok"}
```

Notes:
- CORS allow_origins includes `http://localhost:3000` explicitly (the Phase 2 Next.js origin) plus `"*"` for dev convenience, per CONTEXT.md discretion decision.
- allow_credentials=False is correct when using `"*"` wildcard alongside named origins.
- `prefix="/api"` produces `/api/dogs` and `/api/dogs/{id}` paths.
- Router is imported from `routers.dogs` (relative import works when uvicorn is run from v1/backend/).
- Health endpoint at /health is a free diagnostic.

Start command (documented in README): `uvicorn main:app --reload --port 8000`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → FastAPI | GET requests arrive from browser or frontend — dog_id path param is untrusted input |
| FastAPI → Supabase | Backend sends queries to Supabase using anon key — no writes, read-only |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | GET /api/dogs/{dog_id} | mitigate | dog_id is passed to `.eq("id", dog_id)` as a parameterized filter — supabase-py uses parameterized queries, not string interpolation; malformed UUIDs return empty list → 404 |
| T-02-02 | Information Disclosure | CORSMiddleware | accept | allow_origins includes `"*"` for local dev; this is intentional for internal tooling (per CONTEXT.md); no credentials are exposed in responses |
| T-02-03 | Denial of Service | GET /api/dogs (unbounded) | accept | Read-only, single admin user, no public access; v1 has no pagination but the dogs table is small (shelter scale); v2 will add pagination |
| T-02-04 | Spoofing | /api/* routes | accept | No authentication on API routes in v1 — intentional per PRD §6 ("No auth check on the API routes in v1") |
</threat_model>
